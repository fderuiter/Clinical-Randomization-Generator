## 2024-05-11 - Custom Toggle Toggles Accessibility
**Learning:** In Angular templates, when creating custom toggle groups composed of standard `<button>` elements (e.g., Flat vs Grouped view modes), assistive technologies like screen readers do not understand the relationship or state of the buttons without explicit ARIA roles. While they can be clicked, their "selected" state (often managed via CSS classes like `bg-indigo-500`) is invisible to a screen reader.
**Action:** When implementing custom button groups that behave like radio buttons or tabs, explicitly use `role="radiogroup"` (or `tablist`) on the container. Apply `role="radio"` (or `tab`) and bind `[attr.aria-checked]` (or `[attr.aria-selected]`) dynamically to the individual buttons to announce their state correctly.
