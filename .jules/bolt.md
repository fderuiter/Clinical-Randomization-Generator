## 2026-05-11 - Avoid Filtering Large Arrays in Hot Loops Unless Necessary
**Learning:** In the minimization algorithm, filtering the `activePool` array of stratum combinations on every iteration of the subject simulation loop created significant overhead when dealing with large numbers of permutations. Because the filtering condition relies on checking caps that change infrequently (only when a cap is reached), most filtering passes returned the exact same array, wasting O(N) operations repeatedly.
**Action:** Introduce a boolean flag (`poolNeedsFilter`) mapped to the actual state changes (i.e. only setting it to true when a subject assignment causes an intersection count to reach its cap). This short-circuits the O(N) filter operation entirely unless an actual invalidation occurred, drastically reducing overhead in large Monte Carlo simulations.
